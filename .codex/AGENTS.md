# .codex 子代理配置约束

本目录只维护本项目 Codex 子代理配置；项目开发规则以仓库根目录 `AGENTS.md` 为唯一权威。

## 优先级

- 根目录 `AGENTS.md` 管开发流程、架构边界、验证要求与多代理 fan-out 策略。
- `.codex/config.toml` 只管子代理运行边界，例如并发数与递归深度。
- `.codex/agents/*.toml` 只管固定子代理的模型、职责、只读范围与输出格式。
- 不在本目录重复维护业务架构、构建命令、i18n、Semi、知识库或 DAG 细则；需要时引用根目录 `AGENTS.md`。

## 固定策略

- 固定子代理默认使用 `model = "gpt-5.5"`。
- 固定子代理默认保持 `sandbox_mode = "read-only"`。
- 固定子代理不得改文件、安装依赖、写数据库、提交代码或继续派生子代理。
- `.codex/config.toml` 保持 `max_depth = 1`，由主代理统一收敛结论。

## 输出格式

子代理统一用中文返回精简结论：

1. 结论
2. 关键证据：文件、符号、路由、命令或日志
3. 影响范围
4. 风险、缺口或不确定项

避免倾倒长日志；只保留能支撑判断的关键行。

## 跨项目复刻任务

当任务涉及“旧项目 / 另一个项目 / 精准复刻 / 迁移前端功能”时，固定子代理只做只读盘点，不直接给实现补丁。

- 前端探查代理输出旧项目前端功能、路由、页面、组件、状态、权限、i18n 与 API 调用证据。
- 后端探查代理输出旧项目后端入口、服务逻辑、数据模型、权限、租户、审计、作业与外部依赖证据。
- 边界检查代理输出“旧项目契约 → 当前 Atlas 落点”的复刻矩阵差距与风险。
- 矩阵未经用户确认前，任何代理都不得进入实现建议之外的代码修改。
