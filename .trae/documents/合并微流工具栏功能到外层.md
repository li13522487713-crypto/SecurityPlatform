# 合并微流工具栏功能到外层

## 背景

当前微流编辑器存在两个工具栏：
1. **截图一（外层工具栏）**：`MendixMicroflowEditorEntry` 通过 `toolbarPrefix` + `toolbarSuffix` + `MicroflowEditor` 内部工具栏组成
2. **截图二（workbench 工具栏）**：`MicroflowWorkbenchToolbar` 组件，在 workbench 模式下通过 `toolbarMode="external"` 显示

两个工具栏存在功能重复（如保存、校验、运行、发布、Undo/Redo 等），导致 UI 冗余。

## 现状分析

| 区域 | 功能 | 位置 |
|------|------|------|
| toolbarPrefix | 返回资源库、微流名称、状态标签、版本号、发布状态、Latest版本、adapter模式、更新时间 | `MendixMicroflowEditorEntry.tsx:522-537` |

### 外层工具栏（截图一）缺少的关键功能
- 独立的"保存"按钮（当前只有 Autosave 开关）
- "运行" / "调试运行"按钮（当前 MicroflowEditor 内部有但需要暴露到外层）
- "校验"按钮
- "验收 120"按钮
- "发布"按钮（当前在 toolbarSuffix 中，但样式不一致）
- Undo / Redo（当前 MicroflowEditor 内部有）
- 节点工具箱 Tag
- 节点计数 Tag
- 更多菜单（导出 PNG / 自动排版 / 适应视图）

## 目标

将截图二（`MicroflowWorkbenchToolbar`）的所有功能合并到外层工具栏中，然后删除截图二。

**成功标准：**
- 外层工具栏包含截图二中的所有功能按钮和状态指示器
- 删除 `MicroflowWorkbenchToolbar` 组件及其所有引用
- 保持 workbench 模式（`toolbarMode="external"`）正常工作
- build 通过

## 实施方案

### 步骤 1：扩展 MicroflowEditor 外层工具栏

**文件：** `packages/mendix/mendix-microflow/src/editor/index.tsx`

在 `toolbarMode === "internal"` 的工具栏非 focusMode 区域（`editor/index.tsx:6172-6215`）添加以下功能：

1. **保存按钮**：在 Undo/Redo 之后添加，调用 `handleSave`
2. **运行 / 调试运行按钮**：在保存之后添加，调用 `handleTestRun` 和 `startDebugSession`
3. **校验按钮**：在运行之后添加，调用 `handleValidate`
4. **验收 120 按钮**：在校验之后添加，调用 `handleConfigureAllNodeAcceptance120`
5. **发布按钮**：在校验 120 之后添加，调用 `props.onPublish`
6. **节点工具箱 Tag**：在 toolbarSuffix 之前添加，调用 `handleToggleToolbox`
7. **节点计数 Tag**：在更多按钮之前添加，使用 `complexitySummary` 数据
8. **更多菜单**：在 toolbarSuffix 之前添加，包含导出 PNG / 自动排版 / 适应视图

同时添加以下图标导入：`IconCheckCircleStroked`, `IconSend`, `IconBranch`, `IconClock`, `IconTickCircle`, `IconDownloadStroked`, `IconRefresh`

### 步骤 2：更新 MicroflowEditorProps 接口

## 假设与决策

1. **MicroflowEditor 内部已有的按钮（Undo/Redo/Run/Save/Validate）直接暴露到外层工具栏**：不需要重新实现，只需在工具栏 JSX 中添加对应的 Button 元素，复用现有的 handler。

2. **workbench 模式的 `toolbarMode="external"`**：仍然保留此模式，但不需要 `MicroflowWorkbenchToolbar` 组件了。`MendixMicroflowEditorEntry` 的 `toolbarMode` 逻辑需要调整，当 `toolbarMode="external"` 时，外层工具栏仍需显示（因为截图二的功能已合并进来）。

3. **节点工具箱功能**：`MicroflowEditor` 已有 `toggleToolbox` 方法，直接调用即可。

4. **状态 Tag 的展示**：外层工具栏已有部分状态 Tag（dirty/saving/error count），需要补充节点计数、工具箱状态等。
