# Atlas Security Platform 前后端能力对齐审查报告

## 审查范围

- 后端：128 个控制器类，覆盖 13 个功能域
- 前端：123 个 `.vue` 页面，79 条 `pathComponentFallbackMap` 映射，146 条菜单种子
- 最近提交：附件/版本管理、S3/MinIO 存储、元数据关联、ERD、Rollup 汇总、数据源预览等

---

## 问题一：孤儿页面 -- 有完整组件但无导航入口（10 个页面）

以下页面的 Vue 组件和 API 调用均已实现，但 **既不在 `router/index.ts` 静态路由中、也不在 `pathComponentFallbackMap` 中、也不在菜单种子 `menuSeeds` 中**，用户无法通过任何常规导航到达：


| 页面文件                                                  | 功能描述                     | API 已接线 |
| ----------------------------------------------------- | ------------------------ | ------- |
| `pages/visualization/VisualizationCenterPage.vue`     | 可视化概览中心（KPI 卡片、风险列表）     | 是       |
| `pages/visualization/VisualizationDesignerPage.vue`   | 可视化设计器（画布、节点、校验、发布）      | 是（预览占位） |
| `pages/visualization/VisualizationGovernancePage.vue` | 可视化治理（审计分页表）             | 是       |
| `pages/visualization/VisualizationRuntimePage.vue`    | 可视化运行态（实例列表、详情抽屉）        | 是       |
| `pages/lowcode/ProcessMonitorPage.vue`                | 流程监控看板（统计卡片、实例表、轨迹）      | 是       |
| `pages/lowcode/MessageCenterPage.vue`                 | 消息中心（模板/记录/渠道三 Tab CRUD） | 是       |
| `pages/lowcode/AiAssistantPage.vue`                   | AI 助手对话（表单/SQL/工作流生成）    | 是       |
| `pages/ApprovalAgentConfigPage.vue`                   | 审批代理人配置                  | 需确认     |
| `pages/ApprovalDepartmentLeaderPage.vue`              | 部门负责人配置                  | 需确认     |
| `pages/ApprovalTaskPoolPage.vue`                      | 审批任务池                    | 需确认     |


**修复方案**：为每个页面在 `menuSeeds` 和 `pathComponentFallbackMap` 中补全对应条目。

---

## 问题二：后端空架子 -- 有控制器但前端完全无覆盖（11 个控制器）

以下后端控制器有完整 API 实现，但 **前端既无 `api-*.ts` 调用函数，也无对应页面**：


| 控制器                            | 路由                                           | 功能              | 严重程度        |
| ------------------------------ | -------------------------------------------- | --------------- | ----------- |
| `ReportsController`            | `api/v1/reports`                             | 报表定义 CRUD       | 中           |
| `DashboardsController`         | `api/v1/dashboards`                          | 仪表盘 CRUD        | 中           |
| `EvidenceChainController`      | `api/v1/evidence-chain`                      | 等保证据链查询与导出      | **高**（等保要求） |
| `ApiConnectorsController`      | `api/v1/connectors`                          | 外部系统连接器管理       | 中           |
| `EventSubscriptionsController` | `api/v1/event-subscriptions`                 | 事件订阅配置          | 低           |
| `PackagesController`           | `api/v1/packages`                            | 应用包导入/导出/分析     | 中           |
| `OutboxController`             | `api/v1/admin/outbox`                        | Outbox 死信管理与重试  | 低           |
| `DiagnosticsController`        | `api/v1/admin/diagnostics`                   | 权限-菜单关系诊断       | 低           |
| `MultimodalController`         | `api/v1/multimodal`                          | 多模态（视觉/ASR/TTS） | 低           |
| `DynamicMigrationsController`  | `api/v1/dynamic-migrations`                  | 数据表迁移管理（v1）     | 中           |
| `MonitorController`（部分）        | `api/v1/monitor/compliance/evidence-package` | 合规证据包导出         | **高**（等保要求） |


**修复方案**：按优先级逐个补充前端 API 封装和管理页面，等保相关（证据链、合规导出）优先。

---

## 问题三：已实现组件未挂载（3 项）


| 组件                                       | 位置                                       | 状态                                                                          |
| ---------------------------------------- | ---------------------------------------- | --------------------------------------------------------------------------- |
| `AttachmentPanel`                        | `components/common/attachment-panel.vue` | 组件完整（上传/绑定/解绑/版本历史），但 **无任何页面引用**                                           |
| `RollupConfigPanel` + `RelationDesigner` | `components/`                            | Rollup 配置 UI 存在，但 `RelationDesigner` **无任何页面 import**；ERD 保存关系时不含 Rollup 字段 |
| 文件版本历史                                   | `api-files.ts` → `getFileVersionHistory` | 仅在 `AttachmentPanel` 中使用，随面板一起无入口                                           |


**修复方案**：

- 在动态表记录详情页（`DynamicTableCrudPage`）或相关业务表单中挂载 `AttachmentPanel`
- 在 ERD 关系编辑或动态表字段管理中集成 `RelationDesigner`，打通 Rollup 配置

## 问题五：缺少菜单种子的已有静态路由（6 项）

以下页面在 `router/index.ts` 有静态路由，但 `**menuSeeds` 中无对应条目**，需要管理员手动在菜单管理中配置才能在侧栏显示：

## 问题六：存储后端无前端配置入口

后端已实现 `AliyunOssObjectStore`、`MinioObjectStore`、`LocalObjectStore` 三种存储适配器，但 **前端无存储后端选择/配置页面**。当前仅通过 `appsettings.json` 配置，建议在系统设置中增加存储配置 UI（如需管理员运行时切换）。

---

## 已确认正常工作的功能入口

以下最近实现的功能有完整的前端入口，确认可达：

- 元数据关联（MetadataLink）：ERD 设计器 → 点击实体引用图标 → `EntityReferencesDrawer`
- 数据源高级预览：数据源管理 → 行操作「SQL 预览」→ `AdvancedDataPreviewDrawer`
- ERD 画布：应用内数据管理 → ERD 模型设计
- 动态表 CRUD：应用内数据管理 → 表记录操作
- 个人访问令牌：系统管理 → 个人访问令牌
- AI 全局配置：管理配置入口
- 文件上传/下载：多处可用（但附件面板未挂载）

## 修复优先级建议

1. **P0 -- 等保合规**：证据链 + 合规证据包导出（EvidenceChain / Monitor compliance）
2. **P0 -- 补全菜单种子**：登录日志、在线用户、服务器信息、定时任务（运维监控必备）
3. **P1 -- 孤儿页面挂载**：可视化 4 页、流程监控、消息中心、AI 助手、审批配置 3 页
4. **P1 -- 附件面板挂载**：在动态表记录详情中集成 AttachmentPanel
5. **P1 -- AlertPage 筛选修复**：将 `severityFilter` 传入 API
6. **P2 -- 后端空架子补前端**：Reports、Dashboards、Connectors、Packages、DynamicMigrations
7. **P2 -- Rollup 配置入口**：在 ERD 关系编辑中集成 RelationDesigner
8. **P3 -- 低优先级**：Outbox、Diagnostics、Multimodal、EventSubscriptions、存储配置 UI
