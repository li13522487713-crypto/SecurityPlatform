# 前端页面国际化遗漏修复计划

## 调研结论（当前遗漏清单）

- 路由标题国际化覆盖不足：`[src/frontend/Atlas.WebApp/src/router/index.ts](src/frontend/Atlas.WebApp/src/router/index.ts)` 中约 `99` 处 `meta.title`，仅 `14` 处显式 `titleKey`，大量路由依赖中文回退映射，新增页面容易漏翻。
- 路由标题“映射兜底”负担过重：`[src/frontend/Atlas.WebApp/src/utils/i18n-navigation.ts](src/frontend/Atlas.WebApp/src/utils/i18n-navigation.ts)` 维护了 `titleKeyByPath/titleKeyByTitle` 双表，说明页面侧未统一走 `titleKey`。
- 页面层硬编码普遍：`[src/frontend/Atlas.WebApp/src/pages/system/RolesPage.vue](src/frontend/Atlas.WebApp/src/pages/system/RolesPage.vue)`、`[src/frontend/Atlas.WebApp/src/pages/system/PermissionsPage.vue](src/frontend/Atlas.WebApp/src/pages/system/PermissionsPage.vue)` 等 CRUD 页面中 `title/placeholder/按钮/弹窗/校验文案/列名` 大量硬编码。
- 组件层仍有用户可见硬编码：`[src/frontend/Atlas.WebApp/src/components](src/frontend/Atlas.WebApp/src/components)` 下审批、AI、工作流相关组件存在中文文案，部分组件未接入 `useI18n`。
- 语言包存在“多源分散”现状：`[src/frontend/Atlas.WebApp/src/locales/zh.ts](src/frontend/Atlas.WebApp/src/locales/zh.ts)`、`[src/frontend/Atlas.WebApp/src/locales/en.ts](src/frontend/Atlas.WebApp/src/locales/en.ts)`、`[src/frontend/Atlas.WebApp/src/i18n/extra-messages.ts](src/frontend/Atlas.WebApp/src/i18n/extra-messages.ts)`、`[src/frontend/Atlas.WebApp/src/i18n/runtime-messages.ts](src/frontend/Atlas.WebApp/src/i18n/runtime-messages.ts)` 并存，键域边界需要先约束再扩展。

## 修复目标与边界

- 目标：补齐“所有页面用户可见文案”的 i18n 接入，避免新增页面再依赖中文回退。
- 仅改前端，不改后端接口契约；保持现有功能与权限逻辑不变。
- 继续遵循小步闭环：每批次包含“代码改造 + 语言包补齐 + 自测验收”。

## 实施分批（前端 / 后端）

### 前端实施计划

```mermaid
flowchart LR
  auditScan[扫描硬编码文案] --> navRouteFix[修复导航壳层与路由标题]
  navRouteFix --> coreCrudFix[修复系统高频CRUD页面]
  coreCrudFix --> domainFix[修复审批工作流与低代码AI页面]
  domainFix --> lintBuildCheck[Lint与构建校验]
  lintBuildCheck --> regressionCheck[中英切换回归验收]
```



1. **批次A（最高优先级，导航与壳层）**
  - 文件：
    - `[src/frontend/Atlas.WebApp/src/router/index.ts](src/frontend/Atlas.WebApp/src/router/index.ts)`
    - `[src/frontend/Atlas.WebApp/src/utils/i18n-navigation.ts](src/frontend/Atlas.WebApp/src/utils/i18n-navigation.ts)`
    - `[src/frontend/Atlas.WebApp/src/components/layout](src/frontend/Atlas.WebApp/src/components/layout)`
  - 动作：
    - 为所有路由补齐 `meta.titleKey`，减少对中文 `title` 的翻译反查。
    - 精简 `titleKeyByTitle` 兜底用法，保留仅兼容历史路由。
    - 菜单/面包屑/浏览器标题统一走 key。
  - 验收：
    - 切换中英文后，主导航、面包屑、页面 title 均同步变化。
2. **批次B（高优先级，系统管理核心页面）**
  - 文件（首批建议）：
    - `[src/frontend/Atlas.WebApp/src/pages/system/RolesPage.vue](src/frontend/Atlas.WebApp/src/pages/system/RolesPage.vue)`
    - `[src/frontend/Atlas.WebApp/src/pages/system/PermissionsPage.vue](src/frontend/Atlas.WebApp/src/pages/system/PermissionsPage.vue)`
    - `[src/frontend/Atlas.WebApp/src/pages/system/MenusPage.vue](src/frontend/Atlas.WebApp/src/pages/system/MenusPage.vue)`
    - `[src/frontend/Atlas.WebApp/src/pages/system/DepartmentsPage.vue](src/frontend/Atlas.WebApp/src/pages/system/DepartmentsPage.vue)`
    - `[src/frontend/Atlas.WebApp/src/pages/system/PositionsPage.vue](src/frontend/Atlas.WebApp/src/pages/system/PositionsPage.vue)`
    - `[src/frontend/Atlas.WebApp/src/pages/system/TenantsPage.vue](src/frontend/Atlas.WebApp/src/pages/system/TenantsPage.vue)`
  - 动作：
    - 页面接入 `useI18n`，替换 `CrudPageLayout` 传入的硬编码标题/占位符/抽屉标题。
    - 替换表格列名、按钮、`message.`*、`popconfirm`、表单校验文案为 key。
    - 统一 key 命名：`system.<module>.<scene>`（如 `system.roles.createTitle`）。
  - 验收：
    - 所有 CRUD 主路径（查/增/改/删/批量操作）中英切换无中文残留。
3. **批次C（中优先级，审批/工作流/低代码/AI 长尾）**
  - 文件范围：
    - `[src/frontend/Atlas.WebApp/src/pages/Approval*.vue](src/frontend/Atlas.WebApp/src/pages/Approval*.vue)`
    - `[src/frontend/Atlas.WebApp/src/pages/workflow](src/frontend/Atlas.WebApp/src/pages/workflow)`
    - `[src/frontend/Atlas.WebApp/src/pages/lowcode](src/frontend/Atlas.WebApp/src/pages/lowcode)`
    - `[src/frontend/Atlas.WebApp/src/pages/ai](src/frontend/Atlas.WebApp/src/pages/ai)`
    - `[src/frontend/Atlas.WebApp/src/components/approval](src/frontend/Atlas.WebApp/src/components/approval)`
    - `[src/frontend/Atlas.WebApp/src/components/ai](src/frontend/Atlas.WebApp/src/components/ai)`
  - 动作：
    - 按页面流量与业务关键度分组推进，优先改入口页与常用弹窗。
    - 对未接入 i18n 的组件补 `useI18n`，只替换用户可见文案，不改业务逻辑。
  - 验收：
    - 高频页面与关键弹窗切到 `en-US` 后无明显中文硬编码。
4. **语言包与规范收口（与A/B/C并行）**
  - 文件：
    - `[src/frontend/Atlas.WebApp/src/i18n/index.ts](src/frontend/Atlas.WebApp/src/i18n/index.ts)`
    - `[src/frontend/Atlas.WebApp/src/i18n/extra-messages.ts](src/frontend/Atlas.WebApp/src/i18n/extra-messages.ts)`
    - `[src/frontend/Atlas.WebApp/src/i18n/runtime-messages.ts](src/frontend/Atlas.WebApp/src/i18n/runtime-messages.ts)`
  - 动作：
    - 约定新增 key 主要落在 `runtime-messages`（页面运行文案）并保持中英对称。
    - 建立 key 命名规范与模块前缀，避免重复语义不同 key。
  - 验收：
    - 新增 key 在中英文均存在；无明显同义重复 key。
5. **自动化校验与回归**
  - 建议新增只读校验脚本（后续执行阶段实现）：扫描 `pages/components/layouts` 用户可见硬编码，输出告警清单。
  - 执行 `npm run lint`、`npm run build`，并对语言切换做 smoke 回归（登录、导航、系统管理 CRUD、审批入口、AI 入口）。

### 后端实施计划

- 本次需求不涉及后端接口与数据库变更。
- 仅做联调确认：前端 i18n 改造后不影响现有 API 调用、鉴权和错误处理展示。

## 风险与控制

- 风险：一次性大范围替换导致回归（尤其校验/提示文案）。
- 控制：按批次提交，小步验证；每批先改页面壳和入口流程，再改细节弹窗。
- 风险：路由 title 双机制并存引发不一致。
- 控制：先补齐 `titleKey`，再逐步弱化 `titleKeyByTitle` 兜底。

## 完成定义（DoD）

- `zh-CN/en-US` 切换下，主导航、页面标题、表格列名、按钮、弹窗、表单校验、消息提示均可双语展示。
- 高优先级页面（系统管理 + 主入口）无明显硬编码残留。
- Lint/Build 通过，关键页面冒烟回归通过。
