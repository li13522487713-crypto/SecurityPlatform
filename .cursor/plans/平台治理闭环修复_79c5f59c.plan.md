## 第一阶段：上线阻断清零（2 周 Quick Wins）

### Task 1 — 全仓 tenant-scope 修复（P0）

**目标文件：** `[Atlas.Infrastructure/Services/Platform/PlatformServices.cs](src/backend/Atlas.Infrastructure/Services/Platform/PlatformServices.cs)`

修复点清单：

- `PlatformQueryService.GetOverviewAsync`：所有 `CountAsync()` 加 `Where(x => x.TenantIdValue == tenantId)`
- `AppManifestQueryService.GetByIdAsync`：加租户条件 `&& x.TenantIdValue == tenantId`
- `AppManifestCommandService.CreateAsync` 的 AppKey 唯一性检查：加 `&& x.TenantIdValue == tenantId`
- `AppManifestCommandService.UpdateAsync/ArchiveAsync`：按 `Id + TenantId` 双条件更新
- `ExecuteRuntimeTaskActionAsync`：取任务时加租户校验

验收：tenant A 创建对象，tenant B 查询返回 404/空；SQL 日志中任何统计查询必须带 tenant 条件。

---

### Task 2 — 伪完成接口处置（P0）

**目标文件：** `[Controllers/AppManifestsController.cs](src/backend/Atlas.WebApi/Controllers/AppManifestsController.cs)`

处置方案（二选一，按工期决策）：

- **方案 A（快）：** `impact-analysis` 改返回 `501 Not Implemented`，`designer snapshot` GET/PUT 同样返回 `501 + {"message":"该功能正在建设中，预计 v1.x 上线"}`，避免假数据运营
- **方案 B（完整）：** 见第二阶段 Task 5/6 补齐真实实现

建议 2 周内先走方案 A，第二阶段再补方案 B。

### Task 3 — CodeRunner 降权（P0 安全）

**目标文件：** 

- `[Infrastructure/Services/AiPlatform/CodeExecution/SandboxedPythonExecutor.cs](src/backend/Atlas.Infrastructure/Services/AiPlatform/CodeExecution/SandboxedPythonExecutor.cs)`
- AI Workflow CodeRunnerStep 注册处

操作：

- 在 Controller/Workflow 调用 CodeRunner 前加权限检查：仅 `PlatformAdmin` 角色可触发
- 在 `DirectPythonExecutor.ExecuteAsync` 后写审计：记录 `tenantId + userId + 代码长度摘要 + 耗时 + 是否超时 + 是否错误`
- 在 `appsettings.json` 增加 `CodeExecution.Enabled: false` 开关，默认关闭，需显式开启

中长期：启动容器沙箱 PoC（Docker restricted + seccomp），替换 `DirectPythonExecutor`。

### Task 4 — DeprecatedApi 升级为可统计（P1）

**目标文件：** `[Attributes/DeprecatedApiAttribute.cs](src/backend/Atlas.WebApi/Attributes/DeprecatedApiAttribute.cs)`

在 `OnActionExecuting` 中额外调用 `IAuditWriter` 写审计记录：

```
resourceType: "DeprecatedApiUsage"
resourceId: "{httpMethod}:{routeTemplate}"  
action: "Called"
target: JSON { tenant, caller(User-Agent), traceId, replacement }
```

这样无需新建表，直接复用现有审计检索能力。
