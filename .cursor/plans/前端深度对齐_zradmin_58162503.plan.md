# 前端深度对齐 ZrAdmin 改进计划

## 差异总览

通过逐文件对比 ZrAdmin（Vue2 + Vuex + Element UI）与当前 SecurityPlatform（Vue3 + Pinia + Ant Design Vue），发现以下 13 类差异需要修正：

---

## 1. 路由守卫 (`router/index.ts`)

**ZrAdmin 行为：**

- 有 NProgress 加载进度条
- 登录页有 `redirect` 参数回跳
- GetInfo 失败时自动 LogOut 并提示错误
- afterEach 钩子关闭进度条

**当前缺失：**

- 无 NProgress
- 无 GetInfo 异常时的自动登出和错误提示
- 无 `redirect` 查询参数回跳（登录成功后应跳转到登录前页面）

**改动文件：** `[src/router/index.ts](src/frontend/Atlas.WebApp/src/router/index.ts)`

```typescript
// 需要安装 nprogress: npm install nprogress @types/nprogress
import NProgress from "nprogress";
import "nprogress/nprogress.css";

NProgress.configure({ showSpinner: false });

router.beforeEach(async (to) => {
  NProgress.start();
  // ... 现有逻辑 ...
  // 异常时: catch -> userStore.logout() -> message.error -> return login
  // 登录成功回跳: to.query.redirect
});

router.afterEach(() => {
  NProgress.done();
});
```

## 2. 用户 Store (`stores/user.ts`)

**ZrAdmin 行为：**

- Login action 在 store 中完成（调用 API + setToken + commit）
- 存储 `name`、`avatar`、`userInfo` 完整用户对象
- 有 `FedLogOut`（前端静默登出，不调 API）

**当前缺失：**

- Login 在 LoginPage 直接完成，store 无 login action
- 无 `name`、`avatar` 独立字段
- 无前端静默登出

**改动文件：** `[src/stores/user.ts](src/frontend/Atlas.WebApp/src/stores/user.ts)`

```typescript
// 新增 login action
async login(tenantId: string, username: string, password: string, extra?: {...}) {
  const result = await createToken(tenantId, username, password, ...);
  setAccessToken(result.accessToken);
  setRefreshToken(result.refreshToken);
  setTenantId(tenantId);
}
// 新增 name/avatar getter
```

---

## 3. Permission Store (`stores/permission.ts`)

**ZrAdmin 行为：**

- 深拷贝路由数据，sidebar 和实际路由使用不同副本
- 有 `defaultRoutes`、`topbarRouters` 支持顶部导航模式
- 动态路由末尾追加 `{ path: "/:pathMatch(.*)*", redirect: "/404" }`
- `filterChildren` 处理 ParentView 路径拼接

**当前缺失：**

- 未深拷贝，sidebar 和路由共用同一数据
- 无 `defaultRoutes`
- 无 404 catch-all 追加
- 无 ParentView 子路由路径拼接

**改动文件：** `[src/stores/permission.ts](src/frontend/Atlas.WebApp/src/stores/permission.ts)`

---

## 4. 动态路由工具 (`utils/dynamic-router.ts`)

**ZrAdmin 行为：**

- `filterAsyncRouter`：递归处理 Layout/ParentView/loadView
- 叶子节点删除 `children` 和 `redirect`
- `loadView`：`(resolve) => require([\`@/views/${view}], resolve)`
- `filterChildren`：ParentView 子路由路径拼接 `el.path = parent.path + '/' + el.path`

**当前缺失：**

- `componentMap` 映射过少，大量页面无法动态加载
- 未删除叶子节点的空 children
- ParentView 路径拼接缺失
- `import.meta.glob` 的路径匹配需要更宽泛

**改动文件：** `[src/utils/dynamic-router.ts](src/frontend/Atlas.WebApp/src/utils/dynamic-router.ts)`

核心改动：去掉硬编码 componentMap，改为通用 glob 匹配：

```typescript
function resolveComponent(component?: string) {
  if (!component) return;
  if (component === "Layout") return () => import("@/components/layout/RouterContainer.vue");
  if (component === "ParentView") return () => import("@/components/layout/RouterContainer.vue");
  // 通用匹配：先找 pages/{component}.vue，再找 pages/{component}/index.vue
  const candidates = [
    `../pages/${component}.vue`,
    `../pages/${component}/index.vue`
  ];
  for (const c of candidates) {
    if (pageModules[c]) return pageModules[c];
  }
  return () => import("@/pages/NotFoundPage.vue");
}
```

## 5. TagsView Store（新增）

**ZrAdmin 有完整的 `tagsView` store：**

- `visitedViews`：已访问标签列表
- `cachedViews`：keep-alive 缓存列表
- Actions：addView/delView/delOthersViews/delAllViews/updateVisitedView/delRightTags/delLeftTags/delCachedView

**当前状态：** 无 tagsView store，TagsView 组件内部用 ref 管理，不支持 keep-alive。

**新增文件：** `src/stores/tagsView.ts`

---

## 6. TagsView 组件 (`components/layout/TagsView.vue`)

**ZrAdmin 行为：**

- 使用自定义 ScrollPane 横向滚动
- 右键菜单：刷新、关闭当前、关闭其他、关闭左侧、关闭右侧、全部关闭
- affix 固定标签（如首页不可关闭）
- 联动 tagsView store
- 路由变化时自动添加标签并滚动到当前

**当前缺失：**

- 仅用 a-tabs，无右键菜单
- 无 affix 标签
- 无 store 联动
- 无刷新页面功能

**改动文件：** `[src/components/layout/TagsView.vue](src/frontend/Atlas.WebApp/src/components/layout/TagsView.vue)`

---

## 7. MainLayout (`layouts/MainLayout.vue`)

**ZrAdmin 行为：**

- Sidebar + Navbar（含 hamburger + breadcrumb + 右侧工具栏）+ TagsView + AppMain
- AppMain 使用 `<keep-alive :include="cachedViews">`
- 有 hamburger 折叠按钮
- 有 mobile 响应式 drawer 遮罩
- header 有渐变背景色

**当前缺失：**

- 无 hamburger 折叠按钮（虽然有 collapsed 但没有手动切换的按钮组件）
- 无 keep-alive
- 无 mobile 响应式支持
- Breadcrumb 在内容区而非在 header 中
- 通知铃铛、全屏按钮在重构时被移除

**改动文件：** `[src/layouts/MainLayout.vue](src/frontend/Atlas.WebApp/src/layouts/MainLayout.vue)`

---

## 8. SidebarMenu (`components/layout/SidebarMenu.vue`)

**ZrAdmin 行为：**

- 有 Logo 组件
- `SidebarItem` 递归组件，支持任意深度嵌套
- `hasOneShowingChild` 单子路由优化（只有一个子节点时，提升为父级展示）
- `resolvePath` 使用 `path.resolve` 拼接路径
- 处理外链（isExternal）
- el-scrollbar 包裹

**当前缺失：**

- 仅支持两级嵌套（直接 v-for + v-if 判断 children）
- 无递归组件
- 无单子路由优化
- 无路径拼接
- 无外链处理
- 无 Logo

**改动文件：** `[src/components/layout/SidebarMenu.vue](src/frontend/Atlas.WebApp/src/components/layout/SidebarMenu.vue)`
**新增文件：** `src/components/layout/SidebarItem.vue`（递归组件）

## 9. 权限指令 (`directives/permission.ts`)

**ZrAdmin 行为：**

- `v-hasPermi`：检查 `*:*:`* 全权限标识
- `v-hasRole`：检查 `admin` 超级角色
- 两个独立指令

**当前缺失：**

- 无 `*:*:`* 全权限标识检查（直接检查角色名）
- 无 `v-hasRole` 指令

**改动文件：** `[src/directives/permission.ts](src/frontend/Atlas.WebApp/src/directives/permission.ts)`, `[src/main.ts](src/frontend/Atlas.WebApp/src/main.ts)`

```typescript
// v-hasPermi 应检查 *:*:*
const ALL_PERMISSION = "*:*:*";
const has = userStore.permissions.some(
  (p) => p === ALL_PERMISSION || required.includes(p)
);

// 新增 v-hasRole
export const hasRole: Directive<HTMLElement, string[]> = { ... };
```

## 10. 登录页面不变

登录页当前已有的增强（租户选择、CapsLock 检测、冷却机制）是 SecurityPlatform 自身的等保 2.0 特性，优于 ZrAdmin 的简单表单，**保留不变**。但需补充：

- 登录成功后使用 `userStore.login()` 代替直接调 API
- 支持 `redirect` 查询参数回跳

**改动文件：** `[src/pages/LoginPage.vue](src/frontend/Atlas.WebApp/src/pages/LoginPage.vue)`
