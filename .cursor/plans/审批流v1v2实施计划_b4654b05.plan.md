# 审批流（Workflow）V1+V2 完整实施计划

## 0. 现有基础盘点

经过全面代码探索，当前代码库已具备 **非常成熟的审批流基础设施**：

### 后端已有

- **25+ 实体**：`ApprovalFlowDefinition`、`ApprovalProcessInstance`、`ApprovalTask`、`ApprovalNodeExecution`、`ApprovalHistoryEvent`、`ApprovalParallelToken`、`ApprovalCopyRecord`、`ApprovalDelegateRecord`、`ApprovalTaskTransfer`、`ApprovalAgentConfig`、`ApprovalTimeoutReminder`、`ApprovalCommunicationRecord` 等
- **完整枚举**：`FlowNodeType`（12 种节点）、`ApprovalMode`（All/Any/Sequential/Vote）、`AssigneeType`（11 种分配策略）、`ApprovalOperationType`（46 种操作）、`RejectStrategy`、`ReApproveStrategy`、`MissingAssigneeStrategy`
- **核心引擎**：`FlowEngine`、`FlowDefinitionParser`、`AssigneeResolver`、`ConditionEvaluator`、`DeduplicationService`
- **操作处理器**：`TransferOperationHandler`、`AddAssigneeOperationHandler`、`BackToModifyOperationHandler`、`ClaimTaskHandler`、`UrgeTaskHandler`、`CommunicateHandler` 等
- **后台服务**：`ApprovalTimeoutReminderHostedService`、`ApprovalTimeoutAutoProcessHostedService`、`ApprovalExternalCallbackRetryHostedService`
- **7 个 Controller**：`ApprovalFlowsController`、`ApprovalRuntimeController`、`ApprovalTasksController`、`ApprovalCopyRecordsController`、`ApprovalDepartmentLeadersController`、`ApprovalAgentController`、`WorkflowController`
- **20+ Repository** 实现

### 前端已有

- **@antv/x6** 集成（含 dnd 插件、vue-shape）
- **两种设计器视图**：树形编辑器（`ApprovalTreeEditor`）+ X6 画布（`X6ApprovalDesigner`）
- **12 种节点 Shape**：Start、Approve、Copy、Condition、Inclusive、Route、CallProcess、Timer、Trigger、End 等
- **属性面板**：`ApprovalPropertiesPanel`（审批/抄送/条件/路由/子流程/定时器/触发器）
- **composable**：`useApprovalTree`（增删改查节点、校验）、`useApprovalTreeHistory`（撤销/重做）
- **运行时组件**：`BatchTransferDialog`、`CommunicationPanel`、`JumpNodeSelector`、`LfFormRenderer`
- **完整 API 函数**：50+ 个审批流相关 API

### FlowLong 差距分析已完成

[`.cursor/plans/flowlong_gap_analysis`] 和 [`.cursor/plans/flowlong_feature_replication`] 显示所有阶段已标记完成。

### M2: 节点配置与连线增强（1.5 周）

**目标**：完善节点属性配置面板和连线交互，覆盖 PRD 4~6 节要求。

**前端任务**：

- 审批节点面板增强（`ApprovalPropertiesPanel`）：
  - 审批人来源：验证 11 种 `AssigneeType` 在 UI 中的完整配置
  - 审批方式：单人/或签/会签/阈值签的 UI 配置
  - 拒绝策略配置（或签：任一拒绝 vs 全部拒绝；会签：任一拒绝即失败）
  - 空审批人处理策略（阻断/转交管理员/跳过）
  - 操作开关（允许拒绝/退回/转交/加签/意见必填/附件要求）
  - SLA 配置（时限、提醒、催办、升级策略）
  - 节点级字段权限配置（可见/可编辑/必填/隐藏）
- 抄送节点面板：抄送人来源、触发点、可见内容
- 条件分支配置：
  - 简易模式（字段+运算符+值）
  - 条件字段来源（表单字段/发起人上下文）
  - 分支规则（互斥、从上到下匹配、默认分支必须存在）

  - 锚点悬浮显示
  - 可连接目标高亮、不可连接红色提示

- 验证 `Vote` 模式的阈值计算逻辑，补充"不可达阈值"检测
- 验证并行块异常策略在 `FlowEngine` 中的实现完整性

### M3: 并行网关 UX 与校验增强（1 周）

**目标**：实现 PRD 第 5 节和第 10 节的并行块设计器体验和发布校验。

**前端任务**：

- 并行块组合拖入（方式A）：
  - 拖入"并行块"自动生成 Split + Join + 两条分支占位
  - 分支管理：在 Split 属性面板增加/删除/命名分支
- 并行块容器样式：
  - Split/Join 之间浅色背景框
  - 视觉上明确作用域
- 校验增强（前端本地校验 `ApprovalTreeValidator`）：
  - Split 必须有 Join 且唯一匹配（`parallel_group_id`）
  - Join 入边必须全部来自该 Split 分支域
  - 并行块内所有路径必须可到 Join
  - 禁止并行分支直接结束流程
  - 禁止跨并行域连线
- 错误定位体验：
  - 校验失败时顶部弹出错误列表
  - 点击错误 → 画布定位到节点 + 闪烁高亮

**后端任务**：

- 在 `ApprovalFlowsController` 的校验接口中补充并行块完整性校验规则
- 补充校验结果模型（节点 ID + 错误类型 + 错误消息）

## 3. 技术方案要点

### 3.1 设计器画布引擎

继续使用现有的 **@antv/x6**（已集成 `@antv/x6-plugin-dnd`、`@antv/x6-vue-shape`），补充以下 X6 插件：

```
@antv/x6-plugin-snapline    # 对齐线
@antv/x6-plugin-selection   # 框选
@antv/x6-plugin-minimap     # 缩略图
@antv/x6-plugin-clipboard   # 复制粘贴
@antv/x6-plugin-keyboard    # 快捷键
@antv/x6-plugin-history     # 撤销重做
```

### 3.2 并行块数据模型

复用现有 `TreeNode` 体系中的 `ParallelNode` 类型（`[src/frontend/Atlas.WebApp/src/types/approval-tree.ts](src/frontend/Atlas.WebApp/src/types/approval-tree.ts)`），扩展：

- `parallelGroupId`：Split 与 Join 共享的组 ID
- `branchId`：并行分支内节点归属的分支 ID
- `exceptionStrategy`：并行块异常策略（`rejectAll` / `adminIntervene`）

### 3.3 后端校验增强

在现有 `validateApprovalFlow` 接口基础上，在 `[FlowDefinitionParser](src/backend/Atlas.Infrastructure/Services/ApprovalFlow/FlowDefinitionParser.cs)` 中补充并行块校验规则（详见 PRD 第 13 节的完整清单）。

### 3.4 运行态页面

- CRUD 页面复用现有 `CrudPageLayout` + `useCrudPage` 模式
- 流程图预览复用现有 `X6PreviewCanvas`，增加节点状态着色
- 审批操作复用现有 API 函数（50+ 个已封装）
