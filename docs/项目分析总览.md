# Atlas Security Platform 项目分析总览

---

## 1. 项目整体介绍

**Atlas Security Platform（Atlas 安全支撑平台）** 是一套面向 **等保 2.0（GB/T 22239-2019）** 合规建设的安全与基础设施支撑平台。系统采用 **多租户隔离**（`X-Tenant-Id` + 行级过滤）、**Clean Architecture**、**强类型 DTO** 与 **审计可追溯** 等原则，为组织提供身份权限、资产与告警、审批流、工作流、低代码/动态表、应用开通与数据源、运维监控等能力。

**典型使用者**：

- 平台超级管理员：管理租户、应用目录项、全局配置。
- 租户管理员：管理本租户应用开通与数据源配置。
- 业务用户：使用审批流、工作流、资产、告警、低代码等能力。

## 2. AI 智能体：平台内定位、可扩展与快速迭代

### 2.1 定制化定位（在本平台语境下）

在本仓库中，**AI 智能体**不是孤立聊天窗口，而是挂载在 **租户 → 应用工作台（`/apps/:appId/*`）** 下的 **可配置运行单元**：与 **Prompt 模板**、**插件/API 工具**、**工作流定义**、**知识库/数据绑定** 等能力同层治理，统一走 **JWT + `X-Tenant-Id` + RBAC** 与审计轨迹。面向场景包括：业务侧「按应用交付」的智能助手、可编排任务流、以及与低代码页面/运行态联动的智能化增强。

**与审批流/工作流的关系（便于边界清晰）**：

- **审批流**：人为主、制度与权责清晰，强调实例、待办与合规留痕。
- **通用工作流**：偏执行编排与状态机，可被智能体或业务事件调用。
- **智能体**：偏「目标驱动 + 工具调用 + 模型推理」的组合；通过 **配置与版本** 迭代，而非每次改核心业务代码。

### 2.2 未来可扩展能力（架构预留方向）

| 方向 | 说明 |
|------|------|
| **工具与连接器** | 插件、HTTP/API 连接器、Webhook 等与外部系统对接，智能体通过统一工具协议扩展能力边界。 |
| **编排深度** | 与工作流节点、审批回调、定时任务等联动，形成「对话触发流程 / 流程回调模型」的闭环。 |
| **多模型与路由** | 模型配置、变量与密钥治理分层，便于按租户/应用切换供应商或模型版本。 |
| **知识与数据面** | 知识库、动态表/租户数据源只读查询等，在强类型与权限约束下扩展上下文来源。 |
| **观测与治理** | 对话/调用链审计、限流、脱敏与等保证据映射，与现有 `IAuditWriter`、监控能力对齐。 |
| **生态与复用** | 模板市场、Prompt/插件复用、跨应用复制（在权限允许范围内），降低重复建设。 |

### 2.3 快速迭代能力（产品化交付视角）

- **配置优于硬编码**：智能体人设、系统提示、工具列表、工作流绑定等以 **数据与配置** 驱动，减少发版依赖。
- **应用级隔离**：资源落在 **应用工作台** 下，便于团队并行：不同应用独立迭代 Agent/Prompt/插件而不互相污染。
- **契约与版本**：API 保持 `api/v1` 等版本策略；配置项演进可通过 **版本号/发布态**（草稿/发布）控制灰度与回滚。
- **前后端对齐**：TypeScript 严格类型 + `docs/contracts.md` 同步，接口变更可追溯，缩短联调周期。
- **安全默认**：关键写操作延续 **幂等键 + CSRF** 等基线，智能化能力不绕过租户与权限模型，避免「为了快而牺牲合规」。

> 术语与对象边界（Application / Workspace / Agent / Workflow）的细化讨论，可参考 `docs/analysis/object-definition-conflicts.md` 与 `docs/analysis/sec-18-四层资源与权限矩阵.md`，避免后续扩展时概念混用。

## 4. 每个模块的功能点（摘要）

| 模块 | 功能点 |
|------|--------|
| **认证与令牌（Auth）** | 登录发令牌、刷新、登出；与安全中间件、租户校验配合 |
| **用户 / 角色 / 权限 / 菜单** | RBAC、菜单与路由元数据 |
| **部门 / 职位** | 组织架构与岗位基础数据 |
| **MFA** | TOTP 等多因素能力 |
| **会话与登录日志** | 在线会话、强制下线；登录成功/失败记录查询 |
| **资产（Assets）** | 资产台账类 CRUD/查询 |
| **告警（Alert）** | 告警列表与分页查询 |
| **审计（Audit）** | 关键操作查询；业务侧通过 `IAuditWriter` 写入 |
| **审批流（Approval）** | 流程定义与发布；发起实例；待办/已办；抄送；撤回、转办、加签、退回等运行时操作；部门负责人、代理等 |
| **工作流（Workflow）** | `WorkflowDefinition` 管理；`RuntimeExecution` 与启动/挂起/恢复/终止；外部事件等 |
| **动态表与记录** | 动态建表、字段、索引；记录 CRUD；AMIS Schema 输出 |
| **低代码** | 表单定义与设计器、低代码应用与页面、仪表盘/可视化、运行态页面渲染 |
| **租户与应用** | 应用目录、租户开通、运行实例与控制台类能力（资源中心、发布中心、迁移治理等） |
| **项目资产（Projects）** | 工作空间下项目隔离与权限边界 |
| **租户数据源** | 连接配置、探测与消费侧治理 |
| **系统管理** | 字典、系统参数、通知公告、消息、定时任务、服务与流程监控、表格个人视图、文件上传下载、健康检查、安全令牌获取等 |
| **AI 智能体与扩展** | 应用级 Agent 配置与对话、Prompt 模板、插件/API 工具、工作流联动、知识库与搜索、模型配置等（持续演进） |

> 文档中部分 Phase/Case 标记为「部分完成/待联调」，以设计目标与已实现接口面为准；详细追踪见 `docs/plan-功能补齐总览.md`。

## 5. 核心业务流程

### 5.1 访问受保护资源

用户登录 → 获取 JWT（及可选 CSRF）→ 请求携带 `Authorization` + `X-Tenant-Id` → 中间件校验租户与身份 → 按角色/权限策略授权 → 进入业务控制器 → 仓储/服务访问数据库（自动租户过滤）→ 统一响应封装。

### 5.2 多租户应用开通与运行

维护应用目录 → 租户开通 `TenantApplication` → 配置/切换租户应用实例与数据源 → 在低代码工作台设计页面与表单 → 通过运行路由（如 `/r/:appKey/:pageKey`）交付给终端用户。

### 5.3 审批流闭环

设计并发布流程定义 → 业务侧发起实例 → 生成待办任务 → 审批人同意/驳回等 → 抄送与已读 → 必要时撤回、转办、加签 → 关键动作写审计。

### 5.4 工作流闭环

维护工作流定义 → 启动运行实例 → 跟踪 `RuntimeExecution`/事件 → 挂起、恢复、终止或接收外部事件驱动状态迁移（偏通用编排，与审批流并行存在）。

### 5.5 合规与审计

敏感写操作配合幂等与 CSRF → 操作留痕至审计域 → 安全监控与健康检查支撑运维与等保证据链（详见 `等保2.0要求清单.md`、`docs/compliance-evidence-map.md`）。
