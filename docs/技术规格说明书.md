## 一、概述

### 1.1 项目名称

**Atlas Security Platform**（阿特拉斯安全支撑平台）

### 1.2 项目定位

面向等保2.0（GB/T 22239-2019）合规建设的安全支撑平台与基础设施支撑平台，由基础设施支撑组件与安全支撑组件构成，采用 Clean Architecture、多租户隔离与严格安全控制。

### 1.3 目标等级

- **安全等级**：三级（安全标记保护级）
- **标准编号**：GB/T 22239-2019
- **标准名称**：信息安全技术 网络安全等级保护基本要求

### 1.4 目标用户

| 角色 | 说明 |
|------|------|
| 平台超级管理员 | 管理所有租户、应用、全局配置 |
| 租户管理员 | 管理本租户应用开通与数据源配置 |
| 业务用户 | 使用审批流、工作流、资产、告警、低代码等能力 |

## 三、系统架构规格

### 3.1 分层架构（Clean Architecture）

```
Atlas.Core（基础抽象、基类、模型）
    ↑
Atlas.Domain + Module.Domain（领域实体）
    ↑
Atlas.Application + Module.Application（DTO、校验、映射、服务接口）
    ↑
Atlas.Infrastructure（服务实现、仓储、SqlSugar）
    ↑
Atlas.WebApi（控制器、中间件、API 层）
```

**依赖规则**：依赖向内流动，外层依赖内层，内层不依赖外层。

### 3.2 领域边界（Bounded Contexts）

| 领域 | 模块 | 主要能力 |
|------|------|----------|
| Auth | 认证 | JWT、刷新、登出、MFA |
| Users | 用户 | 用户 CRUD、密码策略 |
| Roles | 角色 | 角色、权限、菜单授权 |
| Permissions | 权限 | 权限定义、分配 |
| Menus | 菜单 | 菜单树、路由 |
| Departments | 部门 | 部门树、组织架构 |
| Positions | 职位 | 职位、岗位 |
| Assets | 资产 | 资产台账、查询 |
| Alert | 告警 | 告警查询、分页 |
| Audit | 审计 | 行为审计、查询 |
| Approval | 审批流 | 流程定义、实例、任务 |
| Workflow | 工作流 | 定义、实例、事件 |
| DynamicTables | 动态表 | 动态建表、CRUD |
| Projects | 项目 | 项目域隔离 |
| Apps | 应用 | 租户-应用开通 |
| TenantDataSources | 数据源 | 租户-应用级数据源 |
| Notifications | 通知 | 发布、收件箱 |
| ScheduledJobs | 定时任务 | Cron、启停、执行历史 |
| Monitor | 服务监控 | CPU、内存、磁盘、健康 |
| TableViews | 表格视图 | 个人列配置、密度、分页 |
| Files | 文件 | 上传、下载 |
| DictTypes / DictData | 字典 | 字典类型、字典数据 |
| SystemConfigs | 参数 | 系统参数 |
| LoginLogs | 登录日志 | 成功/失败记录 |
| Sessions | 在线会话 | 会话列表、强制下线 |

### 3.3 多租户架构规格

| 项目 | 规格 |
|------|------|
| 租户标识 | 请求头 `X-Tenant-Id`（GUID） |
| 数据隔离 | `TenantEntity` 基类 + SqlSugar QueryFilter |
| 校验规则 | JWT 租户标识与 `X-Tenant-Id` 必须一致 |
| 扩展能力 | 租户-应用订阅、项目模式、多数据源 |

### 3.4 关键设计模式

| 模式 | 说明 |
|------|------|
| Query/Command 分离 | `I{Context}QueryService` 读、`I{Context}CommandService` 写 |
| Repository | 接口在 Application，实现在 Infrastructure |
| 依赖注入 | 各层 `ServiceCollectionExtensions.cs` 注册 |
| 统一响应 | `ApiResponse<T>` 包装，`PagedResult<T>` 分页 |
| 异常处理 | `ExceptionHandlingMiddleware` 全局捕获，`BusinessException` 业务异常 |
| 幂等 | 关键写接口要求 `Idempotency-Key` |
| Anti-Forgery | 写请求需 `X-CSRF-TOKEN` |

## 四、API 规格

### 4.1 版本控制

- **前缀**：`api/v1`
- **策略**：版本并行，弃用窗口不少于 6 个月

### 4.2 通用请求头

| 请求头 | 必填 | 说明 |
|--------|------|------|
| `Authorization` | 已登录接口 | `Bearer <accessToken>` |
| `X-Tenant-Id` | 全部 | 租户标识（GUID） |
| `Idempotency-Key` | 关键写接口 | 幂等键（UUID） |
| `X-CSRF-TOKEN` | Web 写请求 | Anti-Forgery Token |
| `X-App-Id` | 可选 | 应用标识 |
| `X-Client-Type` | 可选 | WebH5 \| Mobile \| Backend |
| `X-Project-Id` | 项目模式 | 项目标识 |

### 4.3 统一响应模型

```json
{
  "success": true,
  "code": "SUCCESS",
  "message": "OK",
  "traceId": "00-abc123...",
  "data": {}
}
```

### 4.4 分页模型

**请求**：`pageIndex`、`pageSize`、`keyword`、`sortBy`、`sortDesc`

**响应**：

```json
{
  "items": [],
  "total": 100,
  "pageIndex": 1,
  "pageSize": 10
}
```

### 4.5 错误码

| 错误码 | 说明 |
|--------|------|
| SUCCESS | 成功 |
| VALIDATION_ERROR | 参数校验错误 |
| UNAUTHORIZED | 未认证 |
| FORBIDDEN | 无权限 |
| NOT_FOUND | 资源不存在 |
| SERVER_ERROR | 服务端错误 |
| ACCOUNT_LOCKED | 账号锁定 |
| PASSWORD_EXPIRED | 密码过期 |
| IDEMPOTENCY_REQUIRED | 缺少幂等键 |
| IDEMPOTENCY_CONFLICT | 幂等键冲突 |
| ANTIFORGERY_TOKEN_INVALID | CSRF 校验失败 |

### 4.6 RESTful 规范

- 资源名用复数
- 路径表示资源层级
- HTTP 动词表达操作语义（GET/POST/PUT/PATCH/DELETE）
- 禁止路径中使用动词（如 `/create`、`/update`）

## 五、安全规格（等保2.0）

### 5.1 身份鉴别

| 控制点 | 规格 |
|--------|------|
| 认证方式 | JWT Bearer + 客户端证书 |
| 令牌有效期 | 访问令牌 15 分钟，刷新令牌 720 分钟 |
| 密码复杂度 | 至少 8 位，含大小写、数字、特殊字符 |
| 密码过期 | 90 天 |
| 失败锁定 | 5 次失败锁定 15 分钟，30 分钟自动解锁 |
| MFA | TOTP 支持 |

### 5.2 访问控制

| 控制点 | 规格 |
|--------|------|
| 模型 | RBAC（基于角色的访问控制） |
| 实现 | Controller 级 `[Authorize(Roles = "...")]` |
| 默认策略 | 需认证 |

### 5.3 数据保护

| 控制点 | 规格 |
|--------|------|
| 租户隔离 | `TenantEntity` + QueryFilter 行级隔离 |
| 数据库加密 | SQLite 加密（可配置） |
| 备份 | 每日备份，保留 30 天 |

### 5.4 审计日志

| 控制点 | 规格 |
|--------|------|
| 记录内容 | 操作人、动作、目标、IP、User-Agent、时间戳 |
| 保留期限 | 180 天（可配置） |
| 服务 | `IAuditWriter` |

### 5.5 HTTP 安全

| 控制点 | 规格 |
|--------|------|
| HTTPS | 生产环境强制 |
| CORS | 白名单校验 |
| 安全头 | X-Content-Type-Options、X-Frame-Options、CSP 等 |
| XSS | 中间件防护 + 白名单路径 |

### 5.6 幂等与防重放

| 控制点 | 规格 |
|--------|------|
| 唯一键 | `tenant_id + user_id + api_name + idempotency_key` |
| 重复请求 | 返回相同业务结果 |
| 冲突 | 同 key 不同 payload 返回 `IDEMPOTENCY_CONFLICT` |
| 保留 | 按配置 N 小时后过期清理 |

## 六、配置规格

### 6.1 核心配置节

| 配置节 | 说明 |
|--------|------|
| Jwt | Issuer、Audience、SigningKey、ExpiresMinutes |
| Security | EnforceHttps、PasswordPolicy、LockoutPolicy、BootstrapAdmin |
| Idempotency | HeaderName、RetentionHours、CleanupIntervalMinutes |
| Database | ConnectionString、Encryption、Backup |
| Tenancy | HeaderName |
| Cors | AllowedOrigins |
| Snowflake | GeneratorId |
| IdGenerator | DefaultAppId、Mappings |

### 6.2 安全配置要求

- **生产环境**：JWT SigningKey 至少 32 字符
- **密钥管理**：禁止在配置文件中硬编码，使用环境变量或安全存储
- **数据库加密**：通过 `DATABASE_ENCRYPTION_KEY` 环境变量提供密钥

## 七、项目结构规格

### 7.1 后端目录结构

```
src/backend/
├── Atlas.Core/                 # 共享抽象、基类、模型
├── Atlas.Domain/               # 领域基类
├── Atlas.Domain.{Context}/     # 领域实体
├── Atlas.Application/          # 应用基类
├── Atlas.Application.{Context}/ # DTO、校验、映射、服务接口
├── Atlas.Infrastructure/        # 服务实现、仓储、SqlSugar
└── Atlas.WebApi/               # 控制器、中间件、配置、Bosch.http
```

### 7.2 前端目录结构

```
src/frontend/Atlas.WebApp/
├── src/
│   ├── main.ts
│   ├── App.vue
│   ├── layouts/
│   ├── pages/
│   ├── router/
│   ├── services/
│   └── types/
├── vite.config.ts
└── package.json
```

### 7.3 新增功能开发流程

1. 定义实体（Domain）
2. 创建 DTO、校验器、映射（Application）
3. 定义仓储接口（Application）
4. 实现仓储（Infrastructure）
5. 实现 Query/Command 服务（Infrastructure）
6. 注册服务（ServiceCollectionExtensions）
7. 添加控制器（WebApi）
8. 创建 `.http` 测试文件
9. 更新前端（如需要）

## 八、编码规范规格

### 8.1 通用规则

| 规则 | 说明 |
|------|------|
| 零警告 | 构建必须 0 错误 0 警告 |
| 异步 | 所有 I/O 必须 async/await |
| 仓储 | 控制器不得直接访问数据库 |
| 可空类型 | C# 启用 Nullable reference types |

### 8.2 .NET 规范

| 项目 | 规格 |
|------|------|
| 缩进 | 4 空格 |
| 命名 | PascalCase 类型/公开成员，camelCase 局部/字段 |
| 命名空间 | File-scoped 优先 |
| 目标框架 | net10.0 |

### 8.3 Vue/TypeScript 规范

| 项目 | 规格 |
|------|------|
| 缩进 | 2 空格 |
| 组件文件 | kebab-case（如 login-page.vue） |
| 组件名 | PascalCase |
| TypeScript | 严格模式，禁止 any |

### 8.4 前后端约束

- **后端**：禁止反射、dynamic、运行时编译；强类型 DTO；禁止循环内数据库操作
- **前端**：禁止 any、unknown、eval；全量类型标注；搜索下拉默认 20 条，支持远程检索
- **契约**：共享契约集中于 `docs/contracts.md`，修改时同步更新
