# 联调验收清单

## 目的

- 验证平台级与应用级两条链路均可独立闭环：
  - 平台集成模式（PlatformWeb + AppWeb + PlatformHost + AppHost）
  - 应用直连模式（AppWeb + AppHost）
- 确认“可启动、可访问、可登录、可进入运行时页面”。
- 固化联调验收标准，作为回归测试基线。

## 环境准备

- 参考 [`联调双模式启动手册`](./联调双模式启动手册.md) 启动对应模式。
- 默认账号：
  - `tenantId = 00000000-0000-0000-0000-000000000001`
  - `username = admin`
  - `password = P@ssw0rd!`
- AppWeb 支持 `VITE_APP_RUNTIME_MODE`：
  - `platform`（默认）
  - `direct`

## 平台集成模式验收

1. 启动 `PlatformHost(:5001) + AppHost(:5002) + PlatformWeb(:5180) + AppWeb(:5181)`。
2. 访问 `http://localhost:5180`，平台登录成功。
3. 从平台应用管理页打开应用运行时（新窗口打开 AppWeb）。
4. 访问 `http://localhost:5181/apps/{appKey}/login`，应用登录成功后跳转到 `entry -> r/{pageKey}`。
5. AppWeb 菜单加载成功，运行时页面可打开，关键接口返回 200。
6. 点击“返回平台”可回到平台应用管理页。

## 应用直连模式验收

1. 启动 `AppHost(:5002) + AppWeb(:5181)`，并设置 `VITE_APP_RUNTIME_MODE=direct`。
2. 访问 `http://localhost:5181/apps/{appKey}/login`。
3. 登录、刷新令牌、获取当前用户、登出接口均可用：
   - `POST /api/v1/auth/token`
   - `POST /api/v1/auth/refresh`
   - `GET /api/v1/auth/me`
   - `POST /api/v1/auth/logout`
4. 运行时链路可用：
   - `GET /api/v1/runtime/apps/{appKey}/menu`
   - `GET /api/app/runtime/pages/{pageKey}/schema`
5. 组织管理与应用设置链路可用（以下接口在 direct 模式均返回 200）：
   - `GET /api/v2/tenant-app-instances/{id}/roles`
   - `GET /api/v2/tenant-app-instances/{id}/departments`
   - `GET /api/v2/tenant-app-instances/{id}/departments/all`
   - `GET /api/v2/tenant-app-instances/{id}/positions`
   - `GET /api/v2/tenant-app-instances/{id}/organization/workspace`
   - `GET /api/v2/tenant-app-instances/{id}`
6. 设置页写操作可用（携带 `Idempotency-Key` 与 `X-CSRF-TOKEN`）：
   - `PUT /api/v2/tenant-app-instances/{id}`
   - `POST /api/v2/tenant-app-instances/{id}/publish`
   - `POST /api/v2/tenant-app-instances/{id}/start`
   - `POST /api/v2/tenant-app-instances/{id}/stop`
   - `POST /api/v2/tenant-app-instances/{id}/restart`
7. 在无 PlatformWeb 参与时可独立完成登录、组织管理与设置页联调。

## 回归与失败场景

- 密码错误：返回 `UNAUTHORIZED`。
- 账号锁定：返回 `ACCOUNT_LOCKED`。
- 密码过期：返回 `PASSWORD_EXPIRED`。
- 缺少租户头：返回 `VALIDATION_ERROR`。
- 未登录访问受保护路由：应跳转到 `/apps/{appKey}/login`。
- 平台与应用同浏览器并行访问：登录态互不覆盖（`atlas_platform` / `atlas_app` 命名空间隔离）。

## 运行时内核迁移验收清单（第三轮深挖）

### 1. runtime-core 落点核对

- `@atlas/runtime-core` 已承载动作、绑定、生命周期的核心接口与执行入口。
- `app-web/src/runtime/actions/*`、`app-web/src/runtime/bindings/*` 与 `@atlas/runtime-core` 的导入/导出关系已形成单向依赖（app-web 侧无重复实现）。
- `@atlas/designer-vue` 设计器宿主能力通过 `DesignerCanvas` 提供 `mode` 切换与插槽注入。

### 2. 运行链路核对

- AMIS 事件触发能够到达动作执行器并返回 `ActionExecutionSummary`。
- `when` 条件与分支/循环动作在运行态能被正确解析与执行。
- 页面生命周期钩子（`onPageInit/onRouteChanged/beforeSubmit/afterSubmit`）与 `runtime-core` 执行入口一致。

### 3. 契约与文档核对

- `docs/contracts.md` 已补齐 `runtime-core` 与 `designer-vue` 的接口分工说明。
- `docs/app-web-runtime-core-interfaces.md` 已补齐第三轮深挖落点与验收清单。
- 改动文件清单与接口变更在交付说明中闭环记录。

### 4. 兼容与回归

- 现有设计器入口（实体/关系/视图/转换）仍可在 `platform-web` 中打开。
- 关键页面与接口链路在平台模式与直连模式下均可访问并可回放。
- 未启用第三轮新能力前，现网已有运行时页面应保持不降级体验。
