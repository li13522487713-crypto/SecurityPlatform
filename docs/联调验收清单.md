# 联调验收清单

## 目的

- 验证登录 → 主菜单 → 工作流设计器入口的端到端链路。
- 确认权限控制、审计日志与多租户请求头完整。
- 为上线前的等保2.0控制点提供可追溯记录。

## 环境准备

- 后端配置好 JWT SigningKey（生产环境不少于 32 位）。
- 通过环境变量配置 Bootstrap 管理员账号，避免明文落库。
- 前端代理指向后端（Vite `/api/v1` → 后端 `/api`，不做路径重写）。

## 核心验收步骤

1. 启动后端服务并确认数据库完成初始化（含种子数据）。
2. 启动前端并访问登录页。
3. 使用默认管理员账号登录。
4. 进入主菜单并确认“工作流设计器”入口可见。
5. 点击“工作流设计器”进入页面，加载步骤类型成功。
6. 退出登录后访问 `/workflow/designer`，应被路由守卫拦截并跳转到登录页。
7. 调用 `POST /api/v1/auth/logout` 并确认审计记录写入。

## 验收要点

- 登录成功后 `/api/v1/auth/me` 返回 `roles` 与 `permissions`，且包含 `workflow:design`。
- 前端缓存 `auth_profile`，主菜单基于权限渲染。
- 未授权用户访问工作流页面会被拦截。
- 审计日志中包含 LOGIN / LOGOUT / TOKEN_REFRESH 等关键事件。

## 失败场景验证

- 密码错误：返回 `UNAUTHORIZED`。
- 账号锁定：返回 `ACCOUNT_LOCKED`。
- 密码过期：返回 `PASSWORD_EXPIRED`。
- 缺少租户头：返回 `VALIDATION_ERROR`。
