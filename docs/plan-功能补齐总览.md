## 总体原则

- 分层结构：严格遵循 Clean Architecture，Domain → Application → Infrastructure → WebApi。
- 强类型约束：后端 DTO/实体/配置全强类型；前端 TypeScript 全量类型标注。
- 异步优先：所有 Controller/Service/Repository 使用 async/await。
- 租户隔离：数据访问必须带租户边界，跨租户默认不可见。
- 审计可追溯：关键操作必须记录审计日志并支持查询。
- 契约同步：所有接口变更需同步更新 `docs/contracts.md`。
- 文档先行：每个功能先写规格文档，再写代码。

### 产品化重构关键变更基线（与主计划同步）

- 统一元模型：`AppManifest`、`AppRelease`、`RuntimeRoute`、`PackageArtifact`、`LicenseGrant`、`ToolAuthorizationPolicy`、`FlowDefinition`。
- 统一入口：
  - 平台：`/console`
  - 应用：`/apps/:appId/*`
  - 运行：`/r/:appKey/:pageKey`
  - 兼容：`/settings/*`（Deprecated）
- 新增 API 分组统一为 `api/v1`（平台/应用/运行/治理），并采用强类型 DTO 与校验。
- 所有写接口强制 `Idempotency-Key + X-CSRF-TOKEN`，覆盖幂等冲突、跨租户拒绝、敏感字段脱敏契约。

- Gate-R1：完成 GUI 手工全链路测试并形成报告（平台 -> 应用 -> 运行 -> 任务/审批 -> 审计）。
- Gate-R2：契约、`.http`、集成测试、e2e、上线文档与运维手册齐套。
