## 总体原则

- 分层结构：严格遵循 Clean Architecture，Domain → Application → Infrastructure → WebApi。
- 强类型约束：后端 DTO/实体/配置全强类型；前端 TypeScript 全量类型标注。
- 异步优先：所有 Controller/Service/Repository 使用 async/await。
- 租户隔离：数据访问必须带租户边界，跨租户默认不可见。
- 审计可追溯：关键操作必须记录审计日志并支持查询。
- 契约同步：所有接口变更需同步更新 `docs/contracts.md`。
- 文档先行：每个功能先写规格文档，再写代码。

## 路线图基线（2026-03 规划版）

- 产品化重构主路线图以 `docs/plan-产品化重构-12-sprint.md` 为准，按 **Sprint1~Sprint12 + Gate-R1/R2** 推进。
- `docs/plan-平台控制台与应用数据源.md` 作为平台/应用域专题约束文档继续生效（数据源不可变、共享策略、别名等）。
- Gate-R1（强制）：完整 GUI 手工全链路测试（控制台→应用→运行态→动态表→审批→工作流→模板→移动）。
- Gate-R2（强制）：提交 `src/backend/Atlas.WebApi/atlas.db`（`hangfire.db` 视验收要求）。
- 本追踪表中的 Case 状态更新应与 12 Sprint 节奏保持一致（B/F/D/T 同步勾选）。

### 验收标准

**验证码**
- [ ] 登录页展示图形验证码（可点击刷新）
- [ ] 验证码错误时返回明确错误提示
- [ ] 验证码通过 `appsettings.json` 控制开关

- [ ] 勾选"记住我"后 refreshToken 有效期延长为 30 天

- [ ] 可查询当前活跃会话列表（用户名、IP、登录时间、客户端）

**XSS 防范**
- [ ] 所有 POST/PUT 字符串输入经过 HTML 编码处理
- [ ] 白名单路径（富文本接口）可排除
- [ ] 通过 `appsettings.json` 配置排除路径
