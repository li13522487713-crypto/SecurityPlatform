# SEC-18 四层资源与权限矩阵（含 SEC-57/58）

## 1. 任务信息

- Linear：`SEC-18`（`[P1] 重划平台、租户、应用、项目四层权限边界`）
- 覆盖子任务：`SEC-57`、`SEC-58`
- 所属里程碑：`P1 模型与边界收口`

## 2. 四层资源清单（SEC-57）

| 资源 | 主归属层级 | 典型使用方 | 是否允许跨层引用 |
|---|---|---|---|
| 模型目录、连接器定义、插件定义 | 平台层 | 平台管理员 | 允许（只读消费） |
| 组织用户、租户数据源、租户策略 | 租户层 | 租户管理员 | 条件允许 |
| Agent、WorkflowDefinition、Prompt、Knowledge | 应用层 | 应用管理员/开发者 | 允许到运行层 |
| ProjectAsset、项目成员绑定 | 项目层 | 应用管理员/项目负责人 | 仅在应用层内部 |

## 3. 高冲突资源列表

| 资源 | 冲突表现 | 裁决规则 |
|---|---|---|
| 插件 | 平台定义与应用配置混淆 | 平台定义，应用仅安装实例 |
| 数据源 | 租户资源与应用绑定混淆 | 租户拥有，通过 binding 赋权 |
| Workflow | 定义态与运行态权限混淆 | 定义权限归应用，执行权限归运行 |

## 4. 角色权限矩阵（SEC-58）

| 资源类型 | 平台管理员 | 租户管理员 | 应用管理员 | 应用用户 | 开发者 |
|---|---|---|---|---|---|
| 平台治理资源 | 全权限 | 只读 | 无 | 无 | 无 |
| 租户治理资源 | 只读 | 全权限 | 只读 | 无 | 无 |
| 应用装配资源 | 只读 | 只读 | 全权限 | 只读 | 编辑/调试 |
| 运行态资源 | 只读 | 只读 | 运维 | 使用 | 调试回看 |
| 项目资源（可选） | 无 | 只读 | 管理 | 使用 | 编辑 |

## 5. 平台下发与 App 自治边界

| 能力 | 平台下发 | App 自治 |
|---|---|---|
| 模型策略/额度 | 是 | 否 |
| 插件定义版本 | 是 | 否 |
| 应用资源编排 | 否 | 是 |
| Prompt/Workflow 参数化 | 否 | 是 |
| 发布审批策略 | 是 | 触发申请 |

## 6. 授权冲突处理原则

1. 先判层级归属，再判角色权限；
2. 同一资源若出现多层归属冲突，以“主归属层级”裁决；
3. 平台下发策略优先级高于应用自定义策略；
4. 审计日志必须记录“层级+角色+决策结果”。
