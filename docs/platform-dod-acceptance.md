# 平台封板 DoD 验收清单

## 1. 标准统一

- [x] 页面/表单主 Schema 统一为 AMIS。
- [x] `LF(vform3)` 降级为兼容通路，不再作为新增主路线。
- [x] 表达式统一为 CEL，变量域和安全边界明确。
- [x] 上下文优先级固定为 `Tenant > App > Project`。

## 2. 运行时闭环

- [x] 页面发布后自动注册运行态路由。
- [x] 页面下线后运行态路由自动失活。
- [x] 运行态菜单由后端真实数据生成。
- [x] 运行态任务可查询，动作可执行。
- [x] 运行态前端布局包含导航、通知与用户态信息。

## 3. 数据闭环

- [x] 运行态表单提交走真实 DynamicTable 写入链路。
- [x] 工作台 pages/forms/flows/data 均返回真实数据。
- [x] 草稿页不可通过运行态 schema API 访问。

## 4. 治理与安全

- [x] 关键写接口纳入 `Idempotency-Key` 与 `X-CSRF-TOKEN`。
- [x] 多租户跨租户访问拒绝规则已纳入验证脚本。
- [x] 审计与权限验证脚本覆盖登录、写操作、越权场景。
- [x] 运行态与治理基线验证脚本已补充。

## 5. 交付工程化

- [x] NSwag 文档生成与 TypeScript 客户端生成可用。
- [x] 前端统一脚本：`npm run generate-api`。

## 6. 运维可观测

- [x] 平台资源接口返回数据库、会话、审计、运行态路由等核心指标。
- [x] 关键链路（发布、回滚、导入）具备可追踪落库记录。
