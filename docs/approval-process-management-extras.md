# 审批流程管理附加能力差异分析（AntFlow 对照）

## 概述

本文档对比 AntFlow.net 的流程管理附加能力，分析哪些功能需要纳入 Atlas 范围，哪些可以暂缓实现或通过其他方式实现。

## 一、流程类型/分类（Process Category）

### AntFlow 能力
- 支持流程分类管理（如：人事类、财务类、采购类等）
- 分类用于流程列表筛选和分组展示
- 分类可以设置图标、排序等

### Atlas 当前状态
- ❌ **未实现**：`ApprovalFlowDefinition` 没有分类字段
- 流程列表仅支持按名称、状态筛选

### 是否需要实现
**优先级：P2（可选）**

**分析：**
- ✅ **需要实现**：分类功能有助于流程管理和用户体验
- 实现方式：在 `ApprovalFlowDefinition` 中添加 `Category` 字段（字符串或枚举）
- 实现成本：低（添加字段、更新查询接口、前端筛选）

**建议：**
- 先添加 `Category` 字符串字段（灵活，支持自定义分类）
- 后续可按需扩展为分类表（如果需要分类权限、图标等）

## 二、快捷入口（Quick Entry）

### AntFlow 能力
- 支持将常用流程设置为快捷入口
- 用户可以在首页快速发起常用流程
- 支持个人快捷入口和系统推荐

### Atlas 当前状态
- ❌ **未实现**：没有快捷入口功能
- 用户需要从流程列表中选择流程发起

### 是否需要实现
**优先级：P2（可选）**

**分析：**
- ✅ **建议实现**：提升用户体验，特别是对于高频流程
- 实现方式：
  - 方案1：在 `ApprovalFlowDefinition` 中添加 `IsQuickEntry` 布尔字段（系统级快捷入口）
  - 方案2：创建 `ApprovalUserQuickEntry` 表（用户级快捷入口，更灵活）
- 实现成本：中（需要前端配合）

**建议：**
- 先实现方案1（系统级快捷入口），简单快速
- 后续可按需扩展方案2（用户自定义快捷入口）

## 三、流程权限/可见范围（Process Permission/Visibility）

### AntFlow 能力
- 支持设置流程的可见范围（全部用户/指定部门/指定角色/指定用户）
- 支持设置流程的发起权限（谁可以发起该流程）
- 支持流程级别的权限控制

### Atlas 当前状态
- ⚠️ **部分实现**：
  - 流程定义有 `Status`（Draft/Published/Disabled），可以控制是否可用
  - 但没有细粒度的可见范围和发起权限控制
  - 权限控制依赖系统级的角色权限（RBAC）

### 是否需要实现
**优先级：P1（重要）**

**分析：**
- ✅ **需要实现**：多租户场景下，不同部门/角色可能需要不同的流程可见性
- 实现方式：
  - 方案1：在 `ApprovalFlowDefinition` 中添加 `VisibilityScope` JSON 字段（存储可见范围配置）
  - 方案2：创建 `ApprovalFlowPermission` 表（结构化存储，支持复杂权限规则）
- 实现成本：中高（需要权限检查逻辑、前端配合）

**建议：**
- 先实现方案1（JSON字段），快速满足基本需求
- 在查询服务中添加权限过滤逻辑
- 后续可按需扩展方案2（如果需要更复杂的权限规则）

## 四、版本信息（SysVersion）

### AntFlow 能力
- `t_sys_version` 表存储系统版本信息
- 流程定义有版本号，支持版本历史
- 可以查看流程定义的版本变更历史

### Atlas 当前状态
- ✅ **已实现**：`ApprovalFlowDefinition.Version` 字段已存在
- ⚠️ **部分实现**：
  - 版本号自动递增（每次更新时）
  - 但没有版本历史记录（无法查看历史版本）
  - 没有版本回滚功能

### 是否需要实现
**优先级：P2（可选）**

**分析：**
- ⚠️ **可选实现**：
  - 当前版本号已满足基本需求（标识当前版本）
  - 版本历史功能对大多数场景不是必需的
  - 如果需要版本历史，可以通过以下方式实现：
    - 方案1：创建 `ApprovalFlowDefinitionHistory` 表（存储历史版本）
    - 方案2：使用 Git 或版本控制系统（适合复杂场景）
- 实现成本：中（需要历史表、查询接口、前端展示）

**建议：**
- **暂不实现**：当前版本号已足够
- 如果后续需要版本历史，再实现方案1

## 五、其他附加能力

### 1. 流程图标/封面图
- **AntFlow**：支持为流程设置图标或封面图
- **Atlas**：未实现
- **优先级**：P2（可选）
- **建议**：暂不实现，前端可以统一使用默认图标

### 2. 流程描述/说明
- **AntFlow**：支持流程描述、使用说明等
- **Atlas**：未实现
- **优先级**：P2（可选）
- **建议**：可以在 `ApprovalFlowDefinition` 中添加 `Description` 字段（简单实现）

### 3. 流程排序
- **AntFlow**：支持流程列表排序（按使用频率、创建时间等）
- **Atlas**：未实现
- **优先级**：P2（可选）
- **建议**：可以在 `ApprovalFlowDefinition` 中添加 `SortOrder` 字段

## 实施建议

### Phase 1（立即实现 - P1）
1. **流程权限/可见范围**：
   - 在 `ApprovalFlowDefinition` 中添加 `VisibilityScopeJson` 字段
   - 实现权限检查逻辑（在查询服务中过滤）
   - 更新 DTO 和 API

### Phase 2（按需实现 - P2）
2. **流程分类**：
   - 在 `ApprovalFlowDefinition` 中添加 `Category` 字段
   - 更新查询接口支持按分类筛选
   - 前端添加分类筛选器

3. **快捷入口**：
   - 在 `ApprovalFlowDefinition` 中添加 `IsQuickEntry` 字段
   - 更新查询接口支持查询快捷入口
   - 前端添加快捷入口展示

4. **流程描述**：
   - 在 `ApprovalFlowDefinition` 中添加 `Description` 字段
   - 更新 DTO 和 API

### Phase 3（暂不实现）
5. **版本历史**：当前版本号已足够，暂不实现版本历史
6. **流程图标**：暂不实现，使用默认图标
7. **流程排序**：暂不实现，使用默认排序

## 实施优先级总结

| 功能 | 优先级 | 实现方式 | 成本 | 建议 |
|------|--------|---------|------|------|
| 流程权限/可见范围 | P1 | JSON字段 + 权限检查 | 中 | ✅ 立即实现 |
| 流程分类 | P2 | 字符串字段 | 低 | ⚠️ 按需实现 |
| 快捷入口 | P2 | 布尔字段 | 中 | ⚠️ 按需实现 |
| 流程描述 | P2 | 字符串字段 | 低 | ⚠️ 按需实现 |
| 版本历史 | P2 | 历史表 | 中 | ❌ 暂不实现 |
| 流程图标 | P2 | 文件存储 | 中 | ❌ 暂不实现 |
| 流程排序 | P2 | 整数字段 | 低 | ❌ 暂不实现 |

## 结论

**必须实现（P1）**：
- 流程权限/可见范围（多租户场景必需）

**建议实现（P2）**：
- 流程分类（提升用户体验）
- 快捷入口（提升用户体验）
- 流程描述（简单且有用）

**暂不实现**：
- 版本历史（当前版本号已足够）
- 流程图标（非必需）
- 流程排序（非必需）
