# 平台统一 Schema 与表达式规范（封板基线）

## 目标

- 统一平台设计态与运行态规则，避免双栈分裂。
- 降低新增应用的实现成本与维护成本。
- 保证前后端表达式求值一致，满足等保可审计要求。

## 统一 Schema 策略

- 主 Schema：AMIS（页面、表单、运行态渲染统一使用）。
- 兼容策略：`LF(vform3)` 仅用于历史审批流兼容维护，不再作为新增功能主路线。
- 运行态入口：`/r/:appKey/:pageKey` 仅接受已发布页面的 AMIS Schema。

## 表达式策略

- 统一表达式引擎：`CelExpressionEngine`。
- 可用变量域：`form.*`、`record.*`、`user.*`、`page.*`、`app.*`、`tenant.*`、`global.*`。
- 禁止能力：`eval`、反射、危险函数调用。
- 使用范围：
  - 表单联动与可见性
  - 字段校验
  - 审批条件分支
  - 权限显示规则

## 上下文优先级（权威来源）

- Tenant：`X-Tenant-Id` + JWT Claim 双重校验（不一致直接拒绝）。
- App：运行态由路由参数 `appKey` 决定；工作台由 `appId` 决定。
- Project：仅在启用项目模式时参与校验（`X-Project-Id`）。

优先级：`Tenant > App > Project`。

## 发布态与草稿态

- 草稿态（Draft）：
  - 可在设计器编辑
  - 不允许通过运行态 API 访问
- 发布态（Published）：
  - 可通过运行态 API 获取 Schema
  - 可进入运行态菜单与路由
- 下线/归档态：
  - 运行态路由失活
  - 菜单不再展示

## 禁止事项（封板规则）

- 禁止引入第二套页面/表单 DSL。
- 禁止新增绕过 CEL 的独立表达式执行器。
- 禁止将草稿页面暴露到运行态入口。
