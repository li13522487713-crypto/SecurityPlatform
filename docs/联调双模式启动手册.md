# 联调双模式启动手册

## 目标

统一支持以下两条联调链路：

- 平台集成模式：PlatformHost + AppHost + PlatformWeb + AppWeb
- 应用直连模式：AppHost + AppWeb（不依赖 PlatformWeb）

默认测试账号：

- 租户：`00000000-0000-0000-0000-000000000001`
- 用户名：`admin`
- 密码：`P@ssw0rd!`

## 模式一：平台集成模式

### 一键启动

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev-start-integration.ps1
```

### 访问入口

- PlatformWeb：`http://localhost:5180`
- AppWeb 登录页：`http://localhost:5181/apps/dev-app/login`

### 关键链路

1. 在 PlatformWeb 登录并进入应用管理页。
2. 从平台打开应用运行时入口（新窗口打开 AppWeb）。
3. 在 AppWeb 登录后自动跳转 `entry -> r/{pageKey}`。
4. 点击“返回平台”可回到平台应用管理页。

## 模式二：应用直连模式

### 一键启动

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev-start-app-direct.ps1
```

该脚本会自动设置 `VITE_APP_RUNTIME_MODE=direct`，AppWeb 将直连 AppHost（5002）。

### 访问入口

- AppWeb 登录页：`http://localhost:5181/apps/dev-app/login`

### 关键链路

1. 使用租户/账号登录。
2. 成功后进入应用运行时首页（路由 `/apps/{appKey}/r/{pageKey}`）。
3. 刷新页面后保持登录态，菜单可正常加载。
4. 退出登录后再次访问受保护路由会跳回登录页。

## 接口与路由约定

- AppWeb 运行模式通过 `VITE_APP_RUNTIME_MODE` 控制：
  - `platform`（默认）
  - `direct`
- AppHost 提供认证兼容入口（供 AppWeb 双模式统一使用）：
  - `POST /api/v1/auth/token`
  - `POST /api/v1/auth/refresh`
  - `GET /api/v1/auth/me`
  - `POST /api/v1/auth/logout`

## 常见问题排查

1. 端口被占用
   - 检查 `5001/5002/5180/5181` 是否冲突，先停止旧进程再启动。
2. 登录 401 或租户错误
   - 确认请求头携带 `X-Tenant-Id`，并使用默认租户 ID。
3. 菜单为空
   - 确认应用实例已创建并有已发布页面；必要时先在平台发布一个最小页面。
4. 平台模式下访问 `/app-host/*` 失败
   - 检查 PlatformHost 是否启动成功，且 YARP 已加载对应 appKey 的运行时路由。
5. 直连模式仍请求 PlatformHost
   - 确认 AppWeb 启动时环境变量为 `VITE_APP_RUNTIME_MODE=direct`。
